the app

One script. Every morning.

  • 📈Fetches live stock prices — S&P 500, Nasdaq 100, VTSAX + 5 cybersecurity tickers from Yahoo Finance
  • 📰Pulls headlines from TLDR AI, TLDR Infosec, TLDR Startups, TLDR Product
  • 🤖Scrapes the latest AI Daily Brief post from beehiiv
  • 🌍Grabs NPR Up First stories from RSS (Mon–Sat, skips Sunday)
  • ✉️Sends a short notification email with a link → opens full brief in browser
  • 🔒Live brief at daily-brief-app.vercel.app/brief — rate limited to 10 views/day via Upstash Redis

    lesson 1: rss feeds break

    The AI Daily Brief RSS
    returned malformed XML.

    ❌ Tried: https://aidailybrief.beehiiv.com/feed

    RSS parse error: not well-formed (invalid token): line 108, column 66. Every single run.

    ✅ Fixed: Scrape /archive instead

    The archive page embeds a window.__remixContext JSON blob with all post slugs and dates. Fetch the latest post, extract <p> tags. Problem gone.

    lesson 2: gmail is hard to automate

    Three attempts to send an email.

    ❌ Attempt 1 & 2 — Gmail SMTP + App Password

    App Passwords are disabled on many Google Workspace / managed accounts. No workaround.

    ✅ Attempt 3 — Resend API

    Free tier, 100 emails/day, one API key, no domain verification needed to start. One HTTP POST.

    ⚠️ Gotcha 1: Cloudflare 403 error 1010

    Resend's API blocked the request from Vercel. Fix: add a User-Agent header to the request.

    ⚠️ Gotcha 2: First email went to Gmail spam

    onboarding@resend.dev is an unknown sender. Fix: add it to Gmail contacts as "Anusha's Daily Brief" — Gmail trusts contacts and future emails land in inbox.

    lesson 3: deployment has gotchas

    Vercel tried to be too clever.

    Gotcha 1

    We had a requirements.txt (a list of Python packages to install).

    Vercel saw it and tried to run a package manager — but our script has zero external packages, so the file was pointless.

    ✓ Fix: delete requirements.txt

    Gotcha 2

    Without requirements.txt, Vercel hunted for a web app entry point (app.py, main.py…). Couldn't find one. Refused to deploy.

    Our script isn't a web app — it's just a function that runs on a schedule.

    ✓ Fix: one line in vercel.json → "framework": null

    lesson 4: design in local, not in prod

    Iterate on your machine
    until the HTML looks right.

    The loop

  • 1. Run python3 daily_brief_email.py
  • 2. Opens daily_brief_preview.html
  • 3. View in browser — tweak CSS
  • 4. Re-run → refresh → repeat
  • No deploy needed. Instant feedback.

    • What we iterated on

  • Stock ticker — single line → 2-row to fit all tickers
  • Whitespace — reduced padding, gaps, margins throughout
  • 3-column grid — removed row alignment so cards stack flush
  • Section colors — per-feed accent colors
  • Feed content — right text length, first sentence only

    • All CSS lives in branding-guidelines.md — design separate from logic. Script reads and injects it at runtime.

    lesson 4: security from day 1

    Never commit secrets.
    Not even once.

    # ❌ never do this
EMAIL_TO = "you@gmail.com"
API_KEY  = "re_abc123"

# ✅ always do this
EMAIL_TO = os.environ.get("EMAIL_TO", "")
API_KEY  = os.environ.get("RESEND_API_KEY", "")

    Git history is public even after a force-push. If a secret lands in a commit, rotate it immediately.

    DOS protection

    The /brief endpoint on Vercel's free tier is rate-limited to 10 requests/day using Upstash Redis. Counter auto-resets at midnight UTC. Returns 429 when exceeded.

    Env vars in use

  • EMAIL_TO · RESEND_API_KEY
  • CRON_SECRET — blocks unauthorized cron triggers
  • UPSTASH_REDIS_REST_URL
  • UPSTASH_REDIS_REST_TOKEN

    the secret weapon

    the agent's memory.

    What is a .md file?

    Plain text with simple formatting — # for headings, - for lists, **bold**. No special software needed. Readable by humans and AI equally.

    Every .md file in this project is a permanent note the agent reads at the start of every session.

    Files in this project

  • CLAUDE.md — how the app works, rules for the agent
  • branding-guidelines.md — all CSS, design decisions
  • SECURITY.md — what secrets to never commit
  • HOW_TO_BUILD.md — every decision and gotcha

    • Without these files, you repeat yourself every session.
    With them, the agent already knows your project — pick up exactly where you left off.

    under the hood

    How Claude actually does things

    Claude doesn't just write text back — it takes actions using built-in tools. Most are automatic. A few you trigger yourself.

    Reading & editing files

    Happens automatically whenever you ask Claude to change code.

  • Write — creates a new file

    Claude can run your script, push to GitHub, or look something up.

  • Bash — runs a terminal command
  • WebFetch — opens a URL
  • WebSearch — searches the web

    • e.g. "run the script and show me the output" → Claude calls Bash

    Type /skill-name to run a named workflow.

  • /commit — writes a commit message, stages files, commits

    when things go wrong

    Vercel logs are your best friend.

    • 📋 Check Vercel logs first — vercel.com → project → Logs
      Shows full stdout from every run: stock fetches, article counts, errors. This is how we found the Resend 403 and the SMTP auth failure.
    • 🔑 Wrong or missing env vars — most common cause of failures
      Go to Settings → Environment Variables, double-check EMAIL_TO and RESEND_API_KEY. After changing any var, you must redeploy — Vercel bakes them in at build time.
    • ⚠️ Function ran but no email arrived — check spam first
      Gmail filters unknown senders. Add onboarding@resend.dev to contacts. Logs will show ✅ Email sent even if it lands in spam.
    • 🔁 Test without waiting for cron — run from Vercel dashboard, not terminal
      Go to Vercel → project → Deployments → Functions → send_brief → Test. Running from terminal skips the env vars Vercel injected — the deployed function already has them baked in.
    • ⚠️ Use Claude Code, not Claude Chat — easy to mix up
      Claude Chat (claude.ai) has no access to your files — you'd have to paste code in and out manually. Claude Code in VSCode reads your whole project, edits files directly, and runs commands. Make sure you're in the right pane in VSCode, not a browser tab.
